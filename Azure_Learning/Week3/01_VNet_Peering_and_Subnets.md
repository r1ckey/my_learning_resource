# Azure Week 3①：VNet (仮想ネットワーク) の隔絶とピアリング

AZ-104認定試験において、最も難易度が高く配点が高い「第一関門」が、この Virtual Network (VNet) です。
Azure上にあなた専用の「閉ざされた空間（LAN）」を作り出し、外部の人をシャットアウトする「砦の設計」を学びます。

## 1. VNet と Subnet（サブネット）の関係

*   **VNet (Virtual Network)**: クラウド上に広げる巨大な土地。例えば `10.0.0.0/16` という巨大なIPアドレスの塊（65536個の家が建つ土地）を確保します。
*   **Subnet (サブネット)**: その巨大な土地を区切った「町内会」。例えば `10.0.1.0/24`（256個の家）。
    *   VM（サーバー）などのリソースは、VNetの直下には置けません。**必ずどこか特定のSubnetの住所の中に所属させなければ起動しません。**

> [!CAUTION]
> **🚨 Exam Trap (Azure特有の予約アドレス)**
> 「あなたは `/24` のSubnetを作りました。理論上は256個のIPアドレス（0〜255）があります。では、このSubnetには最大で何台のVMを置けますか？」
> 
> *   ❌ 256台 (机上の空論)
> *   ❌ 254台 (一般的なネットワークの常識：最初と最後を抜く)
> *   ⭕️ **251台 (Azureの絶対ルール)**
> 
> Azureでは、ネットワークアドレス(`.0`)、ブロードキャストアドレス(`.255`)に加えて、Azure自身が内部のルーター等で使うために **最初から`.1`、`.2`、`.3` の3つのIPを強制的に没収（予約）しています**。結果、私たちが自由に使えるのは「256 - 5 = 251個」になります。（※超高頻出のひっかけ問題です）。

## 2. VNet Peering（ピアリング：橋を架ける）

Azureのデフォルトでは、**「VNet Aにあるサーバー」と「VNet Bにあるサーバー」は、互いにプライベートIP（社内LAN扱いの安全な通信）で会話することは絶対にできません。** 完全に世界が隔離（アイソレート）されています。

これを繋ぐのが **VNet Peering（ピアリング）** という設定です。

*   互いのVNetのIPアドレス範囲が「被っていない（重複していない）こと」が絶対条件です（Aが10.0.0.0で、Bも10.0.0.0だと通信パニックになるため繋げません）。
*   ピアリングの設定は「双方向」から行う必要があります。（AからBへ橋をかけ、BからもAへ橋をかける）。片方だけだと一方通行すらできません。

### 🌉 推移的なルーティングの罠 (Non-transitive)

> [!IMPORTANT]
> **🚀 Exam Focus (推移性：友達の友達は友達か？)**
> 
> **[状況]**: 東京支社のVNet（A）と、大阪支社のVNet（B）をピアリングで繋ぎました（A ⇄ Bは通信可能）。次に、大阪支社のVNet（B）と、福岡支社のVNet（C）もピアリングで繋ぎました（B ⇄ Cは通信可能）。
> 
> **[問題]**: では、東京支社（A）のパソコンから、福岡支社（C）のサーバーに直接リモートデスクトップ接続できるでしょうか？
> 
> **[絶対の正解]**: **通信できません！（到達不可）**
> 
> Azureのピアリングは **非推移的（Non-transitive）** です。「Aの友達(B)の友達(C)は、Aにとっては赤の他人」という冷酷なルールがあります。もしAからCへも直接通信させたい場合は、**別途【A ⇄ C のピアリング設定】をもう一つ追加で結ばなければならない**、というのが試験の最重要法則です。
