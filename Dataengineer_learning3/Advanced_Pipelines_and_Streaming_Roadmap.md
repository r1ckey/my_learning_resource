# Part 3：高度なパイプライン設計とストリーミング ロードマップ

「夜間のバッチ処理」から抜け出し、「今その瞬間の意思決定」を可能にする**リアルタイム処理の果てしない壁**と戦うアーキテクチャ領域です。
さらに後半では、無秩序になりがちなSQLの羅列を、**『ソフトウェア・エンジニアリングの厳格なレベルにまで引き上げる』現代最強のモダンデータスタック（DataOps・dbt）**の思想を解き明かします。

## 🎯 この分野のミッション：なぜモダンデータスタックが爆発的に流行っているのか？
「ストリーミング処理は絶対に止まらない（システムが壊れやすい）」＋「SQLコードのバージョン管理や自動テストがない（コピペで誰が直したか分からない）」。
この地獄のような属人化を排除し、**1つのソースコードで全てをクリーンに管理（Git統合・自動テスト・インフラの自動化）する** ことがシニアエンジニアの必須スキルです。

1. **Week 1: バッチか、ストリーミングか（中庸の「マイクロバッチ」）**
   * 「限界のある（終わりのある）データ」と「無限に降り注ぐ水（イベントストリーム）」の違いと、処理難易度の劇的な差。
2. **Week 2: アーキテクチャ紛争（Lambda vs Kappa）**
   * 遅延と正確性を両立しようと苦しんだ「Lambda（コードの二重管理コスト）」と、現代の「すべてをストリーム（Kafka）に見立てる Kappa」の歴史。
3. **Week 3: 全データの「品質保証書（Data Quality）」と dbt の魔法**
   * ELT（抽出・ロード）させた後の変換（T）において、なぜ世界中の企業が **dbt (data build tool)** を熱狂的に導入しているのか？依存関係とテストの自動生成。
4. **Week 4: DataOps と CI/CD（Terraformとデータ革命）**
   * データの世界にもGitHub Actionsが到来。「Terraform」によるInfrastructure as Codeで、DatabricksやSnowflakeの環境構築をたった1行のコマンドで全自動建設（破壊）する魔法。
