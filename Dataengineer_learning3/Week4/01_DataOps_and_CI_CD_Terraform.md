# Core Week 4①：すべてをコードで支配する「DataOpsとCI/CD (Terraform)」

Week 3の dbt によって「SQLプログラム（データ変換）」がソフトウェアのように綺麗に（Gitで）管理されるようになりました（DataOpsの始まり）。
すると、エンジニアはさらなる高みを望み始めます。**「じゃあ、プログラム（SQL）だけでなく、そのSQLを動かす【クラウドのインフラ環境（DatabricksのクラスターやAWSのバケツ）の設定】そのものも、マウスで手動ポチポチするのをやめて、全部コード（文字列）で管理してしまえないか？」**

これが、クラウド時代のデータ基盤を統治する **Infrastructure as Code (IaC)** と **Terraform（テラフォーム）** の概念です。

## 1. ポータル画面（GUI）の手動構築が死を招く理由

あなたが「よし、明日から新しいデータプロジェクト（PJ-A）を始めるから、1. AWS(Azure)のストレージを作り、2. Databricksのワークスペースを作り、3. エンジニアAさんとBさんに管理者権限を付与しよう」と設計したとします。

*   **[手動構築（GUIポチポチ）の限界]**:
    半日かけてブラウザ上で設定完了しました。しかし半年後、「なんか謎の権限エラーで動かなくなった。誰か設定いじった？」となった場合、
    **「誰が・いつ・どの画面で・どういうボタンの設定を変えたのか」を証明する履歴（証拠）がどこにも残っていないため、永遠に修復できなくなります（環境のブラックボックス化）。**

## 2. Infrastructure as Code の覇者「Terraform」

HashiCorp社が開発した、クラウドインフラを構築するための「呪文（コード / HCL記法）」が **Terraform** です。

```hcl
# 魔法の設計図（リソースの宣言）
resource "databricks_workspace" "my_project_workspace" {
  name     = "PJ-A-prod-workspace"
  location = "japaneast"
  sku      = "premium"
}

resource "databricks_user" "analyst_a" {
  user_name = "sato@company.com"
}
```

この「設計図のテキストファイル（.tf）」を書き、コマンドラインから **`terraform apply (適用しろ！)`** とエンターキーを叩いた瞬間。

Terraformは「今のAzure上にそのDatabricksがあって、佐藤さんのアカウントはあるか？」をクラウドAPIを通して自動監視（差分チェック）し、無ければ勝手にAzure上（1時間かかるGUI操作）に**「数分で、完璧な手順で一寸の違わず、リソースを全自動で建設（構築）」**してくれます。

*   もし佐藤さんが退職した時は、Git上の `resource "databricks_user" "analyst_a"` の5行のコードを「BackSpaceキーで消去（削除）」してコミットするだけです。Terraformは「コードから消えたなら、クラウド上の実体も消滅させるべきだ」と判断し、クラウド側のアカウント削除コマンドを全自動で叩いてくれます。

## 3. DataOpsの極致「データの CI / CD」

Terraform（インフラのコード化）と、dbt（クエリのコード化）が完了すると、世界のトップ企業で採用されている「究極のパイプライン（CI/CD）」が組めるようになります。

> [!IMPORTANT]
> **🌟 Cloud Data Platform における「神々のCI/CD」**
> あなたが、新しい分析SQL（モデル変更）を GitHub（Pull Request）にPushした瞬間、以下の魔法が全自動で起きます。
> 
> 1.  **[CI起動]**: GitHub Actions（ロボット）が反応し、Terraformを叩いて、**クラウド上に完全に空っぽの「あなた用の一時的なDatabricks実験環境と、空っぽのデータベース（スキーマ）」を勝手に建設（Create）**します。
> 2.  **[クローンの錬成 (Zero-Copy Clone)]**: その空っぽのDBに、本番環境の「500TBの本物のデータ」を、魔法（Snowflake等のゼロコピークローン技術：実体を動かさずメタデータだけで一瞬でコピーを作る機能）を使って、「0.01秒」でアタッチ（接続表示）させます。
> 3.  **[テストの断頭台]**: その用意された無菌室の一時環境で、あなたが書いた変更SQLと `dbt test` を走らせます。「よし、エラーは出ないな。売上の集計もバグっていないな」と確認します。
> 4.  **[証拠隠滅とデプロイ]**: テストが通ったらロボットは「このコードを本番環境（Prod）にマージデプロイ」し、先ほど勝手に建設した「一時的な実験環境のインフラ・DB（ストレージ）」を、Terraformの `destroy` コマンドで**跡形もなく全削除（更地）して課金を停止させます。（ここまでわずか10分）。**
> 
> 人間が本番環境（データベース）を直接触ってバグらせる余地を「0%」にし、すべてをソフトウェアのコードテストとロボット（CI/CD）の自動化に委ねる。これが現代データエンジニアリングの『最終形態：DataOps』です。
