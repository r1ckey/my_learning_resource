# Azure Week 2①：最強の運び屋「Azure Data Factory (ADF)」とIR兵器

Azure上でビッグデータを動かす際の**絶対的な「指揮官」であり「運び屋（ETLツール）」が、Azure Data Factory (ADF) です**。

ADF自体にはデータを貯める箱（ストレージ）の機能はありません。「パイプ」と「バルブ」を配置して、「朝5時になったら、A地点（SQL）にあるデータを、B地点（ADLS）に全自動コピペしろ」というスケジュールとオーケストレーションを司ります。

## 1. 影の立役者：Integration Runtime (IR) とは何か？
「A地点からB地点へデータを運ぶ」ためには、当然それを実行するための「PC（サーバー・計算エンジン）」が必要です。ADF内でそのサーバー（計算リソース）の役割を果たすのが、**Integration Runtime (IR)** です。

> [!IMPORTANT]
> **🚀 Exam Focus (3つのIRの使い分け)**
> AZ-104およびDP-203試験で、エンジニアの手腕を試す最大の山場が「ネットワーク要件に合わせた正しいIRの選定」です。
> 
> ### ① AutoResolve / Azure IR (完全マネージド)
> *   **特徴**: Microsoft側が用意しているパブリッククラウド上のサーバー（サーバーレス）。
> *   **使い道**: 「AWSのS3」から「AzureのADLS Gen2」へデータを移すなど、**「インターネット上から見えている（公開・パブリックな）場所同士」**でデータを橋渡しする時に使います。（※一番楽でお気に入りです）。
> 
> ### ② Self-Hosted IR (自己ホスト型IR：スパイ部隊)
> *   **特徴**: **あなたが自社のオンプレミス（社内ビル）のWindows Server等に、手動でインストールして潜ませる**スパイ用のIRソフトウェアです。
> *   **使い道**: クラウド(Azure)からは、「社内の1階の奥にある古いSQL Serverのデータ」なんてネットワーク的（ファイアウォール越し）に絶対に見えません。しかし、このスパイソフト（SHIR）を社内PCに仕込んでおけば、SHIRが内側から社内SQLのデータを盗み出して（読み取って）暗号化し、HTTP通信で外側のAzureへと一方通行でデータを押し上げて（Pushして）くれます。
> *   **「オンプレミスのDB・VNetの奥の安全なDB」という単語が出たら、100%この Self-Hosted IR が正解の選択肢になります。**
> 
> ### ③ Azure-SSIS IR (レガシーの救済)
> *   **特徴**: 昔懐かしいWindowsの「SQL Server Integration Services (SSIS)」の古いパッケージ（`.dtsx` など）を、クラウド上でそのまま動かすため**だけ**に存在する専門IRです。「SSISをリフト・アンド・シフトで移行したい」と出た時のみの正答です。

## 2. パイプライン・アクティビティの依存関係

ADFのキャンバス画面で箱（アクティビティ）を並べて繋ぐ時、その線（依存関係・制約）には4つの色（状態）があります。

1.  **緑色 (On Success):** 前の箱が「成功」したら、次の箱へ進む。（超基本）
2.  **赤色 (On Failure):** 前の箱がエラーで「失敗」した時だけ、この箱（エラーメール送信など）へ進む。
3.  **灰色 (On Completion):** 成功しようが失敗しようが、前が「終わったら」絶対に実行する。
4.  **青色 (On Skip):** 前の箱が（条件分岐などで）実行されずスキップされた時に実行する。

これらを組み合わせ、「もしエラーが起きたらITチームにアラートを出しつつ、後続のDatabricksジョブは走らせない（赤色）」といった美しいコントロールフローを設計します。
