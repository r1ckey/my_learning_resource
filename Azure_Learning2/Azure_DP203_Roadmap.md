# DP-203 Data Engineering on Microsoft Azure 攻略ロードマップ (後半戦)

インフラ編（AZ-104）で構築した「セキュアで絶対に落ちないサーバーとネットワーク」の上に、ペタバイト級のビッグデータを処理するためのパイプラインを敷設（ふせつ）する、Azureデータエンジニアの頂点 **【DP-203】** の領域に入ります。

このパートは、あなたがこれまで学んできた「Databricksの知識」と完全にリンク（直結）する最強のシナジーを生み出します。

## 🎯 DP-203: 試験の概要と配点
「ストレージにどうやって安全にデータを貯めるか（Data Lake）」「それをどうやって加工・移動させるか（ADF・Databricks）」「最終的にどこからクエリを叩かせるか（Synapse）」という、モダンデータスタックの全体像が問われます。

1. **データストレージの設計と実装 (40-45%)** -> *(Week 1で攻略)*
   * ADLS Gen2、パーティション戦略、Cosmos DB等
2. **データ処理の設計と開発 (25-30%)** -> *(Week 2で攻略)*
   * Azure Data Factory (ADF)、Mapping Data Flows、ストリーミング
3. **データセキュリティの設計と実装 (10-15%)** -> *(Week 3, 4で合流)*
   * セキュリティ（暗号化、マスキング）、Private Endpoint通信
4. **データストレージとデータ処理の監視と最適化 (10-15%)** -> *(全体を通して)*
   * Synapse SQL、パーティション最適化、Log Analyticsログ分析

## 🚦 学習の進め方
この `Azure_Learning2` フォルダでは、「ADLS Gen2への直アクセス」や「ADFによるオーケストレーション」といった、Databricksの外周（外側）で発生するAzure特有のETLワークフローを中心に学んでいきます。
「オンプレミスのMySQLから、どうやってDatabricksの入り口（ADLS）までデータを全自動で引っ張ってくるか」というアーキテクチャの答えがここにあります！
