# Core Week 3①：データのソフトウェア化革命「dbt」とデータ品質（Data Quality）

クラウドDWHの普及により、ELT（とりあえずDWHの中に全部データを放り込んでから、DWHの中でSQLで「Ｔ：加工」する）が主流になったと学びました。
しかし、その結果、データアナリストたちが**「DWHの中で、数千行の誰も理解できない巨大なSQL（ストアドプロシージャ）を直接書き散らかし、属人化して（退職したら誰も直せない）、一切テストもバージョン管理（Git）もされない」** という最悪の地獄を生み出しました。

この「無法地帯化したSQL（データ変換）」に、**ソフトウェア・エンジニアリングの厳格な秩序（テスト・Git連携・モジュール化）** を持ち込んだ救世主が **dbt (data build tool)** です。

## 1. dbt がもたらした「SQLのモジュール化（レゴブロック）」

dbtの世界では、「生データから1発で巨大なSQLを書いてダッシュボードを作る」ことを強く禁止し、すべてを **「Model（ただの1つのSELECT文のファイル）」** に細かく分割します。

前章で学んだ「スタースキーマ」や「Bronze/Silver/Gold」といった層を、1つずつの短いSQLファイルに変えます。

*   **`stg_users.sql` (Bronze→Silverの基礎):**
    ここでは「ユーザーテーブルのNULLを省き、小文字に揃える」だけのSELECT文を書く。
*   **`fct_sales.sql` (Silver→Goldの集計):**
    ここで dbt 最大の魔法 **`{{ ref(...) }}` (リファレンス関数)** を使います。
    ```sql
    -- dbtでの記述例
    SELECT 
        u.user_id, COUNT(s.sales_id) 
    FROM {{ ref('stg_users') }} u     -- ⬅︎ さっきのSQLファイルを『部品』として呼び出す
    JOIN raw_sales s ON ...
    ```

**🌟 魔法の効果（DAGの自動生成）**
あなたが `{{ ref('stg_users') }}` と書いた瞬間、dbtは裏側で**「あ、この `fct_sales` のテーブルを作るには、先に `stg_users` のSQLを走らせて完成させておかないといけないな（依存関係）」**と完璧に理解します。
これにより、何百個のSQLファイルが存在しても、どの順番で実行（Airflow等でのDAG実行）すべきかを、**dbtが全自動で依存関係グラフを引いて解決（コンパイル）**してくれるのです。

## 2. データエンジニアの命「データ品質（Data Quality）テスト」の自動化

dbt が世界を制した最大の理由が、「データのテスト（アサーション）」の自動化です。
ソフトウェア開発では「関数が正しい数値を返すか（Unit Test）」を必ず書きますが、データの世界にはそれがありませんでした。

dbtでは、`schema.yml` という設定ファイルに、英語で「こういうゴミデータが入ってこないかテストしろ」と数行宣言するだけです。

```yaml
version: 2
models:
  - name: stg_users
    columns:
      - name: user_id
        tests:
          - unique        # ⬅︎ [テスト要件] IDが絶対に重複しないこと！
          - not_null      # ⬅︎ [テスト要件] 空っぽ（NULL）が絶対に入ってないこと！
      - name: status
        tests:
          - accepted_values: # ⬅︎ [テスト要件] 'active'か'deleted' の文字以外は許さない！
              values: ['active', 'deleted']
```

コマンドで `dbt test` と打つと、dbtが上記のテスト要件を「SQLの `SELECT COUNT(*)` チェッククエリ」に全自動で翻訳してデータベースに投げ、「もし重複したユーザーIDが1件でも見つかったら、パイプラインの実行を『エラー（赤色）』として即座に緊急停止させる」という挙動をしてくれます。

これにより、**アナリストのダッシュボードに「バグった（水増しされた）数字」が表示されて大騒動になるのを、システムの中間層で完璧にブロックして未然に防いでくれる最強の防波堤（Data Quality保証）** が完成しました。
