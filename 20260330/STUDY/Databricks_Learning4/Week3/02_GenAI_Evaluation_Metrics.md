# Part 4 Week 3②：AIをAIに評価させる「LLM-as-a-Judge」

RAGシステムの開発中盤に直面する最大の課題は、**「新しいプロンプト（指示文）に書き換えたけど、以前より賢くなったのか？改悪してバカになったのか？」を【客観的な数値】で測る方法がない** という問題です。

売上予測モデルなら「誤差何円でした（RMSE）」と簡単に採点できますが、AIが書いた「文章（テキスト）」に対して、どうやって点数をつければ良いのでしょうか？

## 1. アノテーター（人間）による評価の限界
昔は、時給を払って数十人の人間を雇い、「AIが1000個の質問に回答した結果の文章を全部読ませて、1点〜5点で採点させる（アノテーション）」しかありませんでした。
しかしこの手法はあまりにコストが高く、数週間かかります。

## 2. LLM-as-a-Judge (裁判官としてのLLM) 実践

これを解決する現代のベストプラクティスが、**「賢いLLM（GPT-4等の強力なモデル）に、別のAIの回答文を読んで採点裁判をさせる」** 手法です。
Databricksでは、**MLflow Evaluate** という機能がこれを完全に内包しています。

### アーキテクトが用意すべき「問題集」
まず、データセットとして数十行〜数百行の「完璧なQA集（例題）」を用意します。

| 質問 (question) | AIが知るべき情報 (context) | 模範回答 (ground_truth) |
| :--- | :--- | :--- |
| 残業時間の申請上限は？ | 第3条：残業は月40時間まで... | 会社の規定では、月の残業時間は最大40時間までです。 |

### MLflowに評価・採点（Evaluate）を依頼する

```python
import mlflow

# 先ほど作成したRAGチェーン（回答AI）に「100問の質問」を解かせ、
# その100個の回答を「Databricksが用意した裁判官モデル」に自動採点させる！
results = mlflow.evaluate(
    model=rag_chain,
    data=eval_dataset_df,            # 先ほどの「問題と模範回答」のリスト
    model_type="question-answering",
    evaluators="default"             # デフォルトでDatabricksの強力なLLM裁判官が採点に動く
)

# 採点結果を出力
print(results.metrics)
```

## 3. なにを採点（評価指標/Metrics）してくれるのか？

Databricksの強力な「評価モデル（LLM-as-a-Judge）」は、以下の様々な観点で **0〜5点の点数（スコアリング）** を自動で弾き出し、ダッシュボードに見やすくグラフ化してくれます。

*   **関連性 (Relevance):** ユーザーの質問に対して、正しく適切な回答をしているか？（的外れなことを言っていないか？）
*   **忠実性 (Faithfulness):** 渡した情報資料（Context）に則って答えているか？（勝手にGoogleで学習した知識で嘘＝ハルシネーション＝を混入させていないか？）
*   **安全性 (Toxicity):** 回答文の中に、攻撃的、暴力的、または差別的な言葉（有毒性）が含まれていないか？

> [!IMPORTANT]
> **🚀 エンジニアの正しいPDCAサイクル**
> プロンプトをAからBに少し書き換えたら、必ずこの `mlflow.evaluate()` を走らせてください。「変更前は関連性スコアが平均 4.1 点だったが、変更後は平均 4.5点に上昇し、ハルシネーション率が0%になった。だから本番環境にデプロイして良い」と、**「勘」ではなく「数値」と「エビデンス」に基づいて**AIを安全にデプロイする。これがGenAIエンジニアの最重要スキルです。
