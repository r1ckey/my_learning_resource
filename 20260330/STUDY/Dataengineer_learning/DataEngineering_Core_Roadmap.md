# Data Engineering 必須知識体系 (ベンダー非依存コア) ロードマップ

DatabricksやAzureといった「ツール」や「クラウドベンダー」固有のボタンの押し方を学んだあなたが、次に向き合うべきは**「なぜそれを設計に取り入れるのか？」という、何十年も変わらないコンピュータサイエンスとアーキテクチャの【原理原則】**です。

Google（GCP/BigQuery）の現場に行こうが、AWS（Redshift）の現場に行こうが、Snowflakeの現場に行こうが、この基礎知識だけは決して変わりません。
技術面接で最も深掘り（Why?）される部分であり、シニアエンジニアになるための登竜門です。

## 🎯 カリキュラムの全体像（全4部構成）

1. **Part 1: データアーキテクチャと分散処理の基礎 (`Dataengineer_learning`)**
   「データをどうやって保存し、どうやって千切って計算するか」の物理的・数学的な歴史とアルゴリズム。
2.  **Part 2: データモデリングとETLパラダイム (`Dataengineer_learning2`)**
   「グチャグチャの生データを、分析しやすい『テーブル（表）』にどうやって変換・設計するか」。システムを長生きさせるモデリングの思想。
3. **Part 3: 高度なパイプライン設計とストリーミング (`Dataengineer_learning3`)**
   「止まらない水流（ストリーミング）」と「バッチ処理」の融合。そしてモダンデータスタック（dbtなど）による自動化。
4. **Part 4: データガバナンスと未来のアーキテクチャ (`Dataengineer_learning4`)**
   データが巨大すぎて会社を脅かす存在になるのを防ぐ「ガバナンス」。そしてData Meshという未来の組織論について。
