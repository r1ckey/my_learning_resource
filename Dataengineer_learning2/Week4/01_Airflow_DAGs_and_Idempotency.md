# Core Week 4：絶対に壊れないパイプライン 「DAG」と「冪等性」

データエンジニアの仕事は「コードを書くこと」ではありません。**「365日、土日の夜中でも決して止まることなく、数1000個のエラーを乗り越えながらデータを加工し続ける『全自動の大工場』を設計すること」**です。

その全自動工場のコンベアの流れ（オーケストレーション）を支配する2つの絶対神、**『DAG（ダグ）』** と **『冪等性（べきとうせい）』** を理解できれば、あなたはどこへ行ってもシニアリードエンジニアです。

## 1. 時間の逆行を許さない「DAG (有向非巡回グラフ)」

もし処理の流れ（パイプライン）の中で、「Aが成功したらBへ行く」さらに「Bの中でエラーが起きたら、もう一度Aに戻ってやり直す（ループさせる）」というコードを書いたとします。
データパイプラインにおいて、この「ループ（無限ループ）」は、朝になり出社すると1万件のデータが数億件に増殖している（重複エラー）という最悪のバグを引き起こします。

そのため、すべてのモダンオーケストレーションツール（Apache Airflow、Mage、Databricks Workflow など）は、**DAG (Directed Acyclic Graph: 有向・非巡回・グラフ)** という数学的モデルを強制採用しています。

*   **Directed (有向):** 処理の矢印は必ず「A ➡️ B」という明確な一方向（未来）に流れる。
*   **Acyclic (非巡回):** 決して後戻り（ループ・逆流）してはならない。BからAに戻る線は絶対に引けない。

このルールにより、「処理がどこまで進んだか（どこで止まったか）」がグラフィカルな盤面に描かれ、運用チームが一発で障害・ボトルネックを特定できるようになります。

## 2. アーキテクトの聖杯「冪等性 (べきとうせい：Idempotency)」

全自動の大工場において、最大の敵は「夜中に一時的なネットワーク瞬断で通信エラーが起きる」ことです。
エラーが起きたら、運用担当者は「あ、止まってる。再実行（Retry / リトライ）ボタンをポチッと押しておこう」とリカバリーを図ります。

> [!WARNING]
> **🚨 死亡するシステムのコード (非・冪等な例)**
> ```sql
> -- 翌日1時00分に、前日の売上をテーブルに入れるバッチ
> INSERT INTO sales_table SELECT * FROM raw_sales WHERE date = 'yesterday';
> ```
> もし上の処理が「80%まで完了して、ネットワークエラーで異常終了」したとします。
> 運用者が「失敗だ！」と思って **再実行（Retry）ボタンをもう一度押した時、どうなるか？**
> さっき書き込めていた80%の行がもう一度上書き挿入され、「売上が1.8倍に重複・水増しされる（データの二重登録バグ）」という大惨事が起きます。これをリカバリーするには、手作業でデータをDELETEして…という地獄の作業が待っています。

**「冪等性（Idempotency）」** とは：
**何回（1回でも、100回エラーで再実行したとしても）、同じ日・同じ引数（パラメータ）で関数やパイプラインを実行したら、最終的なテーブルの結果（状態）が、必ず1回目の成功時と「1ミリも変わらず全く同じ（増減なし）」になる、という設計思想です。**

```sql
-- ✅ プロのデータエンジニアが書くコード (冪等性の保証)

-- 【方法1: いつ再実行されても、対象日時の区画を一度真っさらにしてから入れる】
INSERT OVERWRITE sales_table PARTITION (date = 'yesterday')
SELECT * FROM raw_sales WHERE date = 'yesterday';

-- 【方法2: または、IDが同じなら「更新」、無ければ「新規追加」として吸収させる(UPSERT/MERGE)】
MERGE INTO sales_table AS target
USING (SELECT * FROM raw_sales WHERE date = 'yesterday') AS source
ON target.id = source.id
WHEN MATCHED THEN UPDATE SET * 
WHEN NOT MATCHED THEN INSERT *;
```

**冪等性**が担保されたパイプライン（DAG）であれば、あなた（エンジニア）は夜中の激しい雷雨でシステムエラー通知が来ても、**「あぁ、Airflowに『自動リトライ3回』を組んであって、どうせ冪等だからゴミも残らないし、再実行されても放っておけば治るな」**と、布団の中でそのまま安心して二度寝することができるのです。
