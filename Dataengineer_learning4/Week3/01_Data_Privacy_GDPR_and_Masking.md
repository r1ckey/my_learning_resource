# Core Week 3①：法規制の壁（GDPR）と PII の動的マスキング

DWH（データウェアハウス）に世界中のデータを一極集中させ、「スタースキーマ」や「MDM」によって誰でも簡単に最高の分析ができる「データの民主化」環境が完成しました。
しかし、ここに**「国家の法律」と「ハッカー」**が容赦無く牙を剥いてきます。

## 1. 最凶の法律：GDPR (EU一般データ保護規則) / CCPA

データエンジニアがシステムアーキテクチャを設計する際、絶対に逃げられない法規制が存在します。
その中でも、システムの根幹を破壊しかねない最大の難題が **「忘れられる権利 (Right to be Forgotten)」** です。

*   **要件:** 「私の個人情報を、御社のシステムから全て（完全に）削除してください」と顧客から申請があった場合、企業は**30日以内**に、バックアップやデータレイクの末端に至るまで、その人のデータを「絶対に復元不可能な状態（完全消去）」にしなければなりません。違反した場合は「全世界の年間売上の4%」という会社が倒産するレベルの莫大な罰金が課されます。

> [!IMPORTANT]
> **🚀 なぜ Delta Lake / Iceberg の ACID (UPDATE/DELETE) が革命的だったか？**
> さあ、あなたが「Hadoop（ただのHDFSやS3のCSVの山）」を使った古いデータレイクの管理者だと想像してください。
> そのペタバイトのゴミの山（10万個のCSVファイル）の中から、「山田太郎の行」だけを探し出して、ピンポイントで「削除（DELETE）」できますか？
> ➡️ **答え：物理的に不可能です。**（ファイル全部を再作成しなければならない）。
> 
> ここで、以前学んだ **「データレイクハウス（Databricks / Delta Lake）の偉大さ」** が繋がります。
> SQLで `DELETE FROM users WHERE name = '山田太郎'` と打つだけで、背後でペタバイトのParquetファイルを勝手に整理し、ACIDトランザクションで安全に消し去ってくれる（履歴ログをVacuum機能で完全抹消できる）この機能があるからこそ、大企業は安心してGDPR対応ができる最強のデータ基盤を構築できるのです。

## 2. PII (Personally Identifiable Information) とマスキング

データアナリストは「30代男性の購買傾向」を分析したいだけであり、「山田太郎（東京都渋谷区・電話番号090...）」という**個人を特定できる情報（PII: Personally Identifiable Information）**をダッシュボードで見る必要は一切ありません。

これを見せてしまうと、アナリストのPCを通じて個人情報漏洩事故が起きます。

### 🛡️ 昔の対策（静的マスキング）
「じゃあ、ETLの加工パイプラインで、生データから分析用テーブルに移すついでに、名前と電話番号の列を全部『***』で上書きして、バグった（マスキング済みの）データを作って渡せばいいじゃん！」（静的マスキングのクローン作成）。
➡️ **限界:** 保管するデータ量が2倍になる。そして「人事部長だけは本名と給与額を見たい」と言われた時に「人事専用のテーブル群」をまた別途作らなければならず、運用が破綻します。

### 🛡️ 現代の到達点：動的データマスキング (Dynamic Data Masking: DDM)

現代のクラウドデータベース（Azure SQL、Snowflake、Databricks Unity Catalog）における絶対的スタンダードの手法です。

*   **仕組み:** ストレージに保存されているデータは「本物の山田太郎」のまま、一切変更・破壊しません。
*   **マジック:** データをクエリ（SELECT）して画面に表示する【瞬間】に、データベースのエンジンが人間（アクセスしてきた人）の顔（Entra IDの所属グループ）を見て判断します。
    *   **人事部長がSELECT:** 「あなたは人事部ですね。はい、`山田太郎、給与100万円`を表示します」
    *   **一般アナリストがSELECT:** 「あなたは分析部ですね。はい、データベースから読み取った瞬間にフィルターをかけて、`***、給与***` にすり替えて（マスキングして）画面に返します」

これにより、**データは1つ（SSOT）のままで、権限に応じた究極のセキュリティ統制**が可能になりました。（※Databricksでは、これを `Row Filter / Column Mask` 機能として実装しています）。
